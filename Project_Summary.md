**OCR–TTS System**

This project implements an end-to-end pipeline that takes an image as input, extracts text using OCR, and converts the extracted text into speech using a Text-to-Speech (TTS) model.

The system consists of four main components:
1. **OCR Module (Script)**

This module implements a function, run_gemini_ocr, which takes a preloaded image and a configured Gemini client (passed from app.py) as input, and extracts text using the Gemini API (gemini-2.5-flash).

2. **TTS Module (Script)**

This module implements three main functions:
* **preprocess_text_for_tts**

This function orchestrates multiple text cleaning and preprocessing steps. It takes in-memory text from the OCR module as input, processes it, and returns chunked text suitable for TTS. The text is segmented into chunks with a maximum length of 250 characters.

* **generate_audio**

This function takes the chunked text produced by preprocess_text_for_tts, converts it into speech using a preloaded TTS model (Tacotron2) passed from app.py and returns audio files.

* **combine_audio**

This function accepts the audio files generated by generate_audio, concatenate them into a single audio file, and returns the path where it is saved.

3. **Pipeline Module (Script)**

This module implements the ocr_tts function, which orchestrates the OCR and TTS modules into a unified pipeline. This design abstracts multiple function calls into a single interface, simplifying integration within the app.py script.

4. **app.py Script**

This script serves as the main entry point of the system and integrates all components into a Flask-based web application.

* It loads an input image, configures the Gemini client, and passes them to the run_gemini_ocr function for text extraction.
* It initializes the TTS model (Tacotron2) and passes it to the generate_audio function for speech synthesis.
* It implements a user interface (UI) using HTML and Bootstrap, enabling user interaction with the system.

The Flask application was initially tested locally. It was then containerized using Docker and validated in a local containerized environment. The Docker image was subsequently pushed to Docker Hub and deployed on AWS for production use.

**Conclusion**

This project demonstrated the development of a complete OCR–TTS pipeline capable of extracting text from images and converting it into speech. By integrating the Gemini OCR API with a Tacotron2-based TTS model, the system provides an effective solution for automated text-to-audio conversion.

The modular architecture and deployment using Docker and AWS highlight the system’s scalability and production readiness. Overall, the project showcases the practical application of AI in building end-to-end intelligent systems.

**Recommendations**

* Future work could explore fine-tuning or replacing the OCR component to improve accuracy under varying image conditions.
* The TTS pipeline can be optimized for latency and audio quality, particularly for real-time applications.
Implementing asynchronous or queue-based processing would improve scalability and enable handling of concurrent requests.
* Expanding the system to support multilingual OCR and TTS would significantly enhance its applicability.
Additional deployment optimizations, such as load balancing and monitoring, could further improve production performance.
* The system can be integrated into assistive devices such as smart reading aids for visually impaired users, similar to systems like MonReader, enabling real-time conversion of visual text into speech through an OCR–TTS pipeline.
